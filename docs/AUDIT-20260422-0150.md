### P0（语义错误 / 安全 / 阻塞上线）

#### P0-1. 踢人硬删 submissions，与最新 spec 冲突

- **现象**：admin 踢学生出题单后，该学生在题单内的 submissions 全部从数据库消失；
  `/submissions` 全站列表、个人中心"我的提交"均看不到。
- **复现**：
  1. admin 登录，创建题单 A，加入 stu1。
  2. stu1 提交若干（挂题单 A）。
  3. admin 点"踢出"。
  4. `SELECT * FROM submissions WHERE user_id=stu1.id AND problem_set_id=A.id` = 0 行。
- **根因**：`backend/internal/handlers/admin.go:495-498`
  ```go
  if err := tx.Where("problem_set_id = ? AND user_id = ?", psid, uid).
      Delete(&models.Submission{}).Error; err != nil { return err }
  ```
  最新 prompt 明确要求："`/submissions` 全站仍可见他的原始提交，仅不再贡献题单排名"。
- **影响**：历史数据不可恢复；审计链路断裂；若学生被误踢，rejoin 后历史记录无法重建。
- **修复**：
  - 删除 admin.go:495-498 的 `Delete(&models.Submission{})` 一行。
  - 同步删除 admin.go:484-494 的 `AITask` 字符串反查删除块（保留审计记录）。
  - 在 `handlers/ranking.go:serveProblemset`（283-332）的 raw SQL 增加
    anti-join ban 表：`AND NOT EXISTS (SELECT 1 FROM problem_set_bans b
    WHERE b.problem_set_id = submissions.problem_set_id AND b.user_id = submissions.user_id)`。
  - 同步改 `ranking.go:computeAKPerUser`（192-228）、`handlers/problemset.go:105-128`
    （acByPS / topByPS raw SQL）、`problemset.go:325-332` headline leader。
- **验证**：seed stu10 被预封禁 + 有历史提交 → `/problemsets/3?tab=ranking` 不出现
  stu10；`/submissions?user_id=stu10` 仍然能看到他的提交。

#### P0-2. `/problems` 列表对未登录用户强制跳转

- **现象**：匿名访问 `http://<host>/problems` → 被路由 guard 推到 `/?next=/problems`。
- **复现**：`localStorage.removeItem('liteoj.token')` → 访问 `/problems` → 观察到跳转。
- **根因**：`frontend/src/router/index.ts:52-57` 仅允许 `meta.public === true` 的路由匿名访问；
  当前只有 `/` 标了 public，`/problems` 没标。
- **但后端 `/api/problems` 已用 `OptionalAuth`**（`main.go:99`）——后端准备好匿名访问，
  前端守卫却拦截，产品上不一致。
- **影响**：匿名用户无法浏览题目目录，与 spec A5 "但 `/problems`（列表）允许未登录
  查看" 冲突。
- **修复**：`router/index.ts:10` `{ path: 'problems', ... }` 加 `meta: { public: true }`；
  复测 `ProblemList.vue` 在 `user.isLoggedIn=false` 时仍能工作（my_status 为空即可）。
- **验证**：匿名访问 `/problems` → 看到题目列表，无 `next=` 跳转；点某题 → 跳 `/?next=/problems/:id`。

### P1（正确但性能 / 稳定性有隐患）

#### P1-3. SQLite PRAGMA 未启用

- **现象**：`db/db.go:18-33` 只做 `gorm.Open`，没设 `journal_mode=WAL /
  synchronous=NORMAL / busy_timeout=5000 / foreign_keys=ON / temp_store=MEMORY`。
- **根因**：初版代码未注入。
- **影响**：
  - `queue_workers=1` 下 DELETE 模式 journal 会触发整库锁，并发读写抖动。
  - 大量短写入（submissions + ai_tasks 同时落盘）可能触发 `SQLITE_BUSY`。
  - 无 `foreign_keys` 时外键约束实际不生效（虽然当前 schema 未显式定义 FK，但
    保守开启）。
- **修复**：`db/db.go:Open` 的 sqlite 分支返回前 `db.Exec("PRAGMA ...")` 串行注入。
  失败 `log.Printf` 不 panic。
- **验证**：启动日志打印 5 条 PRAGMA 结果；`sqlite3 ./data/liteoj.db "PRAGMA journal_mode;"`
  返回 `wal`。

#### P1-4. 题单成员/封禁在 ranking/list SQL 无 anti-join

- 见 P0-1 联动。`problemset.go:105-128`、`ranking.go:192-228, 283-332` 全部需要加
  ban 表 NOT EXISTS 子句。若保留当前"硬删"，P0-1 的 fix 不落地则此项 moot；
  但 spec 已确定保留 submissions，此项必须改。
- **修复**：上面 P0-1 一并处理。

#### P1-5. `Submissions` 表无复合索引，题单排名 SQL 全表扫

- **现象**：`ranking.go:283-296` 查询
  `WHERE problem_set_id = ? AND problem_id IN ? AND verdict IN (...)` 目前仅能用
  `problem_set_id` 单列索引（`submission.go:27`）。
- **影响**：题单学生数 > 几百 + 每人数十条提交时，单次 ranking 请求需要扫上千行
  + Go 内存聚合。
- **修复**：`models/submission.go` 加
  `gorm:"index:idx_ps_problem_verdict_created,priority:1"` 等。AutoMigrate 会自动建。

#### P1-6. AI 任务 subject 按字符串反查

- **现象**：`admin.go:487-493` 用 `subject LIKE "submission #<id>"` 反查 AITask。
- **影响**：P0-1 修完后这段代码不再必要，应移除；若将来需要按 submission 反查，
  应在 `models.AITask` 加 `SubmissionID uint` 索引列。
- **修复**：本轮只移除这段（不删 AITask）；`SubmissionID` 列做 backlog。

## 5. 数据库结构优化建议

### 5.1 当前 schema 总览

表：`users / tag_groups / tags / problems / testcases / problem_sets /
problem_set_items / problem_set_members / problem_set_bans / submissions /
ai_tasks / home_pages`。

### 5.2 需新增的索引

```go
// submissions
UserID       uint `gorm:"index:idx_user_created,priority:1;..."`         // 已有
ProblemID    uint `gorm:"index:idx_problem_user,priority:1;..."`         // 已有
ProblemSetID *uint `gorm:"index:idx_ps_pid_verdict_created,priority:1"`  // 新增复合
ProblemID    uint  `gorm:"... index:idx_ps_pid_verdict_created,priority:2"`
Verdict      string `gorm:"index:idx_ps_pid_verdict_created,priority:3"`
CreatedAt    time.Time `gorm:"index:idx_ps_pid_verdict_created,priority:4"`

// problem_set_items
uniqueIndex:uniq_ps_problem(problem_set_id, problem_id)
index:idx_ps_order(problem_set_id, order_index)

// tags
uniqueIndex:uniq_group_name(group_id, name)

// users
LastSeenAt  *time.Time `gorm:"index"`

// ai_tasks
index:idx_status_started(status, started_at DESC)
```

### 5.3 PRAGMA（启动期注入）

```go
db.Exec("PRAGMA journal_mode=WAL")
db.Exec("PRAGMA synchronous=NORMAL")

db.Exec("PRAGMA foreign_keys=ON")
db.Exec("PRAGMA temp_store=MEMORY")
```

3. 删 `./data/liteoj.db` / `.db-wal` / `.db-shm`。
4. 启后端；seed 重建。

  保留代码用于老库升级。
